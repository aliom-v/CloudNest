# CloudNest 使用与运维指南

## 1. 日常怎么用

CloudNest 的日常使用可以分成两部分：

- 前台上传和拿链接
- 后台管理和清理资源

## 2. 前台上传流程

### 上传步骤

1. 打开首页
2. 选择一张 JPG、PNG、WEBP 或 GIF 图片
3. 点击上传
4. 等待返回结果
5. 复制你需要的链接格式

### 上传后你会拿到什么

- 图片预览
- `Public ID`
- 原图 URL
- Markdown 链接
- HTML 链接

### 最近上传记录有什么用

- 方便回看刚上传的图片
- 方便跳到后台做删除联调
- 适合个人场景下快速重复使用

## 3. 后台管理流程

### 登录后台

1. 打开 `/admin`
2. 输入白名单邮箱
3. 输入管理员密码
4. 登录成功后读取资源

### 后台能做什么

- 读取 Cloudinary 真实资源
- 根据关键词搜索资源
- 按格式筛选
- 分页加载更多
- 删除单个资源
- 批量删除资源
- 查看删除审计

## 4. 推荐操作习惯

### 个人使用建议

- 上传目录保持固定，例如 `cloudnest/uploads`
- 默认标签保持简单，例如 `imagebed,mvp,web-upload`
- 先通过首页上传，再通过后台清理无用图片
- 重要图片尽量不要只依赖本地最近上传记录

### 删除建议

- 删除前先核对 `Public ID`
- 批量删除前先用筛选条件缩小范围
- 对不确定的资源先打开原图确认
- 如果你依赖 CDN 缓存刷新，删除时保持勾选 `invalidate`

## 5. 审计怎么理解

删除审计的目的不是替代数据库，而是保证你至少能知道：

- 谁删了资源
- 删了哪张图
- 删除发生在什么时候
- 删除结果是成功、失败还是取消

### 存储策略

- 本地开发：回退到 `.data/delete-audit.jsonl`
- 生产环境：推荐 Vercel Blob

如果你未来想做更强查询、长期留存或多人管理，建议把审计迁移到数据库或日志系统。

## 6. 排错顺序

当你遇到问题时，建议按下面顺序排查。

### 上传有问题

1. 先看首页环境状态卡片
2. 检查 Cloudinary 相关公共变量
3. 确认你当前走的是 `signed` 还是 `unsigned`
4. 再去看浏览器请求和 Cloudinary 配置

### 后台有问题

1. 先确认管理员会话能创建
2. 再确认 Cloudinary 服务端密钥是否完整
3. 然后检查资源读取
4. 最后再测试删除和审计

### 部署后有问题

1. 先确认环境变量是否真的保存到了当前环境
2. 重新触发部署
3. 再做最小链路验证，不要一次测试所有功能

## 7. 安全建议

当前项目适合个人版和小规模使用，但如果你准备长期公开使用，建议尽快补这些能力：

- 限流
- 验证码或机器人防护
- 更正式的管理员认证
- 更细的审计与告警
- 固定依赖版本

## 8. 推荐的操作清单

### 本地开发前

```bash
npm install
npm run verify
```

### 发布前

```bash
npm run verify
```

### 发布后

1. 上传一张图
2. 复制 Markdown 链接
3. 登录 `/admin`
4. 读取资源列表
5. 做一次删除测试
6. 检查审计是否写入

## 9. 你接下来最可能要做的事

### 如果你只是个人使用

- 保持当前结构即可
- 优先关注是否稳定上传和删除

### 如果你准备继续打磨产品

- 补更完整的认证
- 收紧上传策略
- 增强筛选和分页体验
- 把审计迁移到更适合查询的存储

如果你准备接手继续开发，下一步建议阅读 [开发维护指南](./developer-guide.md)。
