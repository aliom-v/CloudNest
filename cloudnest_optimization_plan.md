# CloudNest 优化文档

## 一、背景

在当前版本中，项目已经具备上传、后台读取、删除和审计能力，但围绕目录边界、签名上传暴露面、后台搜索语义和自动化验证仍存在明显缺口。为了避免继续在不稳定基础上叠功能，这一轮先聚焦把关键边界收紧，把行为语义和实现对齐，再补上基础验证闭环。

## 二、本轮要解决的问题

### 2.1 删除范围校验不够严格

当前删除逻辑只使用 `startsWith(uploadFolder)` 判断 `publicId` 是否在允许目录下，这会导致前缀碰撞问题。例如允许目录是 `cloudnest/uploads` 时，`cloudnest/uploads-archive/demo` 也会被误判为可删除。

### 2.2 签名上传接口暴露策略不明确

当前 `POST /api/sign-cloudinary-params` 在配置了服务端密钥后即可被直接调用，缺少明确的“公开签名上传”开关，也没有把“公开可用”和“仅管理员或受控访客可用”区分开。

### 2.3 后台搜索语义和实现不一致

当前后台资源列表接口是先读取一页资源，再在这一页数据上内存过滤。这会让用户误以为自己在搜索整个上传目录，但实际上只是筛选当前页结果。

### 2.4 自动化验证覆盖不足

当前仓库只有 `lint`、`typecheck` 和 `build`，缺少对关键边界逻辑的自动化测试，也没有统一的一键验证入口。

## 三、本轮优化目标

- 删除只允许命中配置目录本身或其子路径，不允许前缀碰撞越界。
- 签名上传必须变成显式策略，而不是“只要配了密钥就默认公开”。
- 后台搜索应当成为真正的服务端搜索，而不是伪搜索。
- 为关键安全/边界逻辑补最小可用的自动化测试，并提供统一验证命令。

## 四、实施决策

### 4.1 目录边界

- 增加上传目录规范化逻辑，统一去掉首尾斜杠和空白。
- 增加“目录本身或其子路径”判断函数。
- 删除接口统一复用该判断函数。

### 4.2 签名上传策略

- 新增环境变量 `ALLOW_PUBLIC_SIGNED_UPLOAD`。
- 默认策略改为：未显式开启时，不对匿名访客公开 signed upload 参数。
- 当 `ALLOW_PUBLIC_SIGNED_UPLOAD=true` 时，才允许首页公开优先使用 signed upload。
- 当公开 signed upload 开启时，接口仍增加同源请求校验，避免被外站直接跨站调用。
- 当公开 signed upload 未开启时，签名接口仅允许管理员会话访问。

### 4.3 后台搜索

- 后台资源读取改为：无查询词时走原有分页列表；有查询词时走 Cloudinary Search API。
- 搜索范围限定在配置的上传目录下。
- 搜索结果保留当前前端展示字段，不改变页面结构。

### 4.4 自动化验证

- 增加 Node 内建测试，覆盖：
- 上传目录规范化与目录边界判断。
- 签名上传公开策略判断。
- Cloudinary 搜索表达式构造。
- 新增 `npm test`。
- 新增 `npm run verify`，统一执行 `lint`、`typecheck`、`test`、`build`。

## 五、本轮不做的事

- 不引入新的第三方测试框架。
- 不重做后台 UI。
- 不在这一轮接入更复杂的限流、验证码或外部认证系统。
- 不处理依赖版本从 `latest` 改为固定版本这一类供应链治理问题，本轮先聚焦运行边界和验证闭环。

## 六、验收标准

- 删除接口对 `cloudnest/uploads-archive/...` 这类前缀碰撞路径返回拒绝。
- 首页只有在显式允许公开 signed upload 时才显示 signed upload 可用。
- 未开启公开 signed upload 时，匿名调用签名接口应返回拒绝。
- 后台带查询词时能够针对整个配置目录进行服务端搜索，而不是只过滤当前页。
- `npm run verify` 能通过。

## 七、执行顺序

1. 先补优化文档。
2. 再收紧目录边界和签名上传策略。
3. 然后接入真正的服务端搜索。
4. 最后补自动化测试和统一验证脚本。
